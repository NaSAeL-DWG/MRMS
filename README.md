这是一个基于 **纯 C++** 开发的终端机房预约系统。

**技术评估：纯 C++，没有技术纯灌垃圾 🗑️**

## 📖 项目背景

本项目为 **黑马程序员 C++ 课程** 的最后一个阶段任务练习作品。主要是为了考察 C++ 面向对象编程、文件读取与写入、STL 容器的综合运用（或者只是为了证明自己真的敲完了这门课）。

## 🛠️ 技术亮点 (如果在稍微认真地审视一遍代码的话)

代码虽然自嘲“纯灌垃圾”，但内部实现上还是应用了现代 C++ 的一些规范特性：
- **内存管理**：大量使用智能指针 (`std::shared_ptr` 和 `std::weak_ptr`) 管理系统中各种对象的生命周期，防止内存泄漏。
- **面向对象设计**：抽象基类 `Identity` 提供多态支持，并派生出 `Student`, `Teacher`, `Admin`。
- **数据持久化**：使用 `fstream` 将各个表单信息（学生、教师、管理员、订单）分离式存储在独立的 CSV 文件中。
- **UI与逻辑分离**：封装了 `UIUtils.hpp` 和 `Login` 处理终端彩色输出、清屏及交互，后台数据统一由 `Provide` 类单例模式提供支持。
- **UTF-8 字符集支持**：通过 `SetConsoleOutputCP(CP_UTF8)` 解决 Windows 控制台默认编码可能引发的中文字符乱码问题。

## 💡 系统功能概览

系统的基础操作基于文本菜单，所有数据实时读写本地的 CSV 格式数据集（如 `students.csv`, `teacher.csv`, `order.csv` 等）：

* **🧑‍🎓 学生端 (`Student`)**
  * 提交机房预约申请（指定机房、周几、上午/下午）。
  * 查看本人的所有预约记录及审核状态。
  * 查看全校所有人的预约记录。
  * 取消本人尚未生效或待审核的预约。

* **🧑‍🏫 教师端 (`Teacher`)**
  * 查看全校所有预约记录。
  * 审批预约申请（通过 / 驳回）。

* **👨‍💻 管理员端 (`Admin`)**
  * 录入新的学生/教师账号信息。
  * 查看系统中已有的所有账号数据。
  * 查看当前各个机房的信息（容量及占用状态）。
  * 一键清空所有预约记录危险操作。
